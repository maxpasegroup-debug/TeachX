# Administrator Guide

Use the administrator account to prepare the institution for daily work.

## First Login

1. Confirm institution settings.
2. Review white-label branding.
3. Create or invite users.
4. Confirm roles and permissions.
5. Review academic year, branch, courses, batches, fee heads, and communication settings.

## Important Areas

- Settings: institution details, colors, templates, PDF branding.
- People: user directory and role overview.
- Communication: announcements, role messages, course messages, batch messages.
- Backup readiness: `/api/backup/readiness` for deployment checks.

Keep one administrator account reserved for recovery access.

## Launch Operations

- Use the Admin Growth OS for platform analytics, support, audit logs, system settings, automation, and reports.
- Review marketplace, AI, commerce, community, and user-management dashboards before public launch.
- Keep payment, video, realtime messaging, and provider integrations disabled until provider onboarding is complete.
- Use `/api/health`, `/api/ready`, and `/api/version` during deployment verification.
